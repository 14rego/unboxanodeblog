  • Design Boiler

    A Boilerplate for Rapid Prototyping

keyboard Develop

Long story short

I built the same basic components in two (soon three) different versions of frameworks. One source set of CSS rules applies to all of the different versions of HTML. Theoretically, these can be used as easily tweak-able examples while rapid prototyping, no matter which framework you choose for your project.

Cutting corners

I was talking with some co-workers and we were discussing - once again - everyone's preferences for front-end theming frameworks. It may seem to designer like starting with Bootstrap or Material is extremely unoriginal, but there a lot of benefits to doing so and I really think they're not embraced nearly enough.

A heard start, sure

First of all, as a person who's designed a web thing or two, I very much appreciate the design coverage. I appreciate being able to change as little or as much as I want to customize my design. But, let's say I don't get all the way down to the details of designing which error color appears on forms, there's already one there that will suffice and I'm covered. When prototyping, this is essential.

Consistency

While it may be a little more difficult to jump in and out of frameworks and keep them all straight, choosing a framework and going with it for an entire project makes it 100% easier for developers to jump in and code without too much ramp-up time. Everyone knows what the code does just by looking at it and they spend a lot less time digging into documentation because they're already familiar with the capabilities and structure.
