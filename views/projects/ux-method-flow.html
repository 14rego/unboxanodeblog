The Big Idea

There are so many user experience methods, it can be hard to keep track of them all. I've compiled data for 50 methods in different research and design categories and presented them in searchable/sortable format. As soon as the questions are set up, you would answer 1-7 questions about where you are in your product or service's journey and it will suggest several of the methods appropriate for your stage in the process. Along with the suggestions are: statistics about each method, procedural tips, further reading resource links, and downloadable PDF worksheets and templates.

Basic React

The build uses Node, React, and Babel for six components (more soon as the question flow builds). So far, there's no need for a state management system like Redux. It's just not that complicated.

I really like the thought of hosting static HTML instead of a dedicated Node server, so my next step may be incorporating Gatsby as soon as it's complete.
