In case you are unfamiliar, the number_format() function in PHP, with a little help in formatting settings, can format your numbers to display commas and decimal points however you see fit. For example, 1000.316 can very easily become $1,000.32 with little-to-no effort.

My first problem with this came while using Chrome input tags of the ‘number’ type. I was pulling info from a a database, formatting it very nicely, then displaying it in the inputs. But in Chrome, a number input will not accept commas, so it simply wasn’t displaying any data at all.

Now, I’m all for making a browser friendly and easy to use, but Chrome adding these extra input features is a little bit annoying. Not to mention, on the date-type inputs, it has a native datepicker - which you have to disable if you already employ them so that you don’t have two pop up at once. I just feel like maybe there should be another html tag for an input that you’d like additional, native options. Maybe a datepicker type? Something.

Side note: some things I can’t believe I didn’t know before: jQuery’s .trigger() - clearly this saved me A LOT of time.
And the cache settings in .ajax() - here I have been hating IE this whole time and now... well, I still hate it, but thanks to ajax’s cache settings, I don’t have much to worry about.
