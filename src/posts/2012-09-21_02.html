So, that was pretty painful, but I think I’m well on my way to getting this mod_rewrite thing figured out. I think the hardest part is understanding the regular expressions and I already had a head start on that, so I didn’t have much of an excuse. What I really needed was:

  1. Find the “physical” .php page, if it exists
  2. If the ‘.php’ extension was left off of the physical file url (if it exists), show the page anyway
  3. If the physical page does not exist, plug the info into the query page
  4. Discern which directory the intended “imaginary” page belongs to so that the proper color scheme can be applied
  5. Find out if the content is in the database and display it in the template, with the color scheme from above

So, believe it or not, this is where the challenge begins. Now, I’d like to get rid of the physical files that I no longer need, but will still need to display the “imaginary” version, where necessary. So, I need to be able to add the extension to an imaginary file that used to be a physical file. Look, boys and girls - this job is not always fun, but I got it done.
