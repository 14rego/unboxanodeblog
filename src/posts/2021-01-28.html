Y’all, GreenSock is still kicking! I used to use GreenSock, like, decades ago when animation started making its way out of Flash. GreenSock was great then because it was lightweight (remember dial-up?!) and the scripting was a lot like ActionScript.

And now, today, I’m re-exporing GreenSock and it’s seemingly countless plug-ins and features for a complicated animation I’m putting together at work. For this one, I’m using the Timeline feature (classic), a newer plugin called ScrollTrigger, and another plug-in called ScrollToPlugin that is pretty minimal.

Timeline

Timeline’s best feature - to me - has always been [the availability of] simplicity. Don’t get me wrong, it can get very complicated, but you create a timeline object, you place tweens on it, and then you traverse it. It’s animation basics and Timeline is all over it. Back in the day, they also had Timeline Lite, which was an even lighter version for the simplest of animations.

ScrollTrigger

The ScrollTrigger plugin is for watching the user’s scroll position and triggering animations based on that position.

It has one really cool feature which it seems I can’t live without for this particular project - it’s called pinning. With pinning, ScrollTrigger creates padding above and below your animation to give additional room for scrolling so that it can "pin" your animation to your viewing frame. So basically the page appears to have stopped scrolling (whizzing by), but now the action of you scrolling is controlling your animation instead. Very clever.

I can only imagine this was developed a couple of years ago when parallax-y web design and development was all the rage and scrolling became more than a way to reach the bottom of the page.

ScrollToPlugin

I actually won’t end up using ScrollToPlugin, it turns out. It’s a quick way of marrying your timeline with the javascript scrollTo() function, which typically finds an object on the page, getting it’s position, and then smoothly scrolling animatedly (is that a word?) to the position.

The reason that’s not useful for me on this project is because my positions are so complicated (thanks to ScrollTrigger), that scrolling the page up and down to a position just isn’t going to cut it. I need horizontal positioning and such, too, which it doesn’t seem to currently offer. Which is fine. It’s fine.

All of that to say… if you’re doing any HTML/JS animations in the near future, GSAP is somehow still where it’s at.
