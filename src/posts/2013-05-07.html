[ Another from the KCDC series… ] I recently went to the Kansas City Developer’s Conference where I like to think I learned a lot and I’d like to share some with you.

Topics discussed included signals, or red flags, that cause you to stop and investigate what could be considered “bad” code.

What I found most interesting:

Feature Envy — when objects and methods are organized poorly, several things will happen. Objects return results from another object. You might look for a magnet for code changes that don’t really apply to this class or object. Or a one-line function returning lots of different things. An object’s functionality should be fully encapsulated.
